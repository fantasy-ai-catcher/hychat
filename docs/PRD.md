# HyChat PRD

版本：0.1  
日期：2026-06-06  
产品形态：Node.js terminal 私密聊天室

## 1. 背景

HyChat 是一个面向小范围朋友群的终端聊天室。用户希望在 terminal 内完成日常聊天，同时能查看共同关注的股票信息。产品重点是轻量、私密、实时、可运行在开发者熟悉的命令行环境中。

## 2. 产品目标

1. 支持我和几个朋友在同一个私密房间里实时聊天。
2. 支持在聊天界面展示共享股票 watchlist 和股票报价摘要。
3. 支持通过命令在 terminal 内管理房间、成员、股票 watchlist 和消息。
4. 使用 Supabase 承担认证、数据库、权限、实时消息和定时刷新能力。
5. 第一版保持 MVP 范围，不做交易、不做投资建议、不做复杂社交关系。

## 3. 非目标

1. 不支持股票下单、模拟交易或投资建议。
2. 不支持公开聊天室、陌生人发现、好友推荐。
3. 不支持移动端、Web 端或桌面 GUI。
4. 不支持语音、视频、文件传输。
5. 不追求毫秒级行情，股票数据以展示和讨论为主。

## 4. 用户角色

### 4.1 房间拥有者

创建房间，邀请朋友，维护房间共享股票 watchlist。

### 4.2 房间成员

加入已授权房间，发送和接收消息，查看在线成员，查看股票信息。

### 4.3 系统任务

定时刷新股票报价缓存，处理第三方股票 API 失败和限流。

## 5. 核心假设

1. 用户规模是小群体，初始目标为 1-3 个房间、每个房间 2-10 人。
2. 用户入口使用昵称 + 邀请码，底层使用 Supabase Anonymous Auth 和 RLS 管理身份。
3. 股票数据源第一版默认使用 Twelve Data，通过可替换的 Stock Provider adapter 接入。
4. 股票查询第一版覆盖美股、港股和 A 股，内部使用统一 symbol 规范隔离不同 API 的代码格式。
5. 股票报价可接受延迟缓存，默认刷新间隔为 60-300 秒，具体取决于外部 API 限额。
6. 如果后续要求港股/A 股盘中实时行情，优先评估 Longbridge 或 Futu 作为替换 provider。
7. 所有私密数据必须通过 Supabase RLS 控制访问。

## 6. 主要功能

### 6.1 账号登录

用户可以在 terminal 中登录、退出，并在重启应用后恢复会话。

功能点：

1. 使用 `/start [nickname] [invite-code]` 激活昵称 profile。
2. 底层通过 Supabase Anonymous Auth 创建匿名用户，不要求 email/password。
3. 本地保存 Supabase session，避免每次启动都重新激活。
4. 支持退出登录并清除本地 session。

验收标准：

1. 未激活用户启动应用时看到 `/start` 引导。
2. 激活成功后能看到自己可访问的房间列表。
3. 退出登录后无法读取任何房间消息。

### 6.2 房间列表和进入房间

用户可以查看自己已加入的房间，并进入指定房间聊天。

功能点：

1. 展示当前用户所属房间列表。
2. 支持进入房间。
3. 支持房间内展示房间名、成员数、在线状态和股票 watchlist。
4. 非成员不能看到房间，也不能订阅房间实时消息。

验收标准：

1. 用户 A 能看到自己加入的房间。
2. 用户 B 未加入该房间时看不到该房间。
3. 用户 B 直接尝试进入房间 ID 时被拒绝。

### 6.3 私密实时聊天

成员可以在同一房间内实时收发文本消息。

功能点：

1. 输入普通文本后发送为聊天消息。
2. 新消息持久化到数据库。
3. 同房间其他在线成员实时收到消息。
4. 新进入房间时加载最近消息历史。
5. 支持消息时间、发送者昵称和正文展示。
6. 支持系统消息，例如成员加入、watchlist 变更、股票刷新失败。

验收标准：

1. 两个 terminal 使用不同账号进入同一房间，用户 A 发送消息后，用户 B 在 2 秒内看到消息。
2. 刷新或重启应用后，最近历史消息仍可见。
3. 非成员无法读取该房间历史消息。

### 6.4 在线状态和输入状态

用户可以看到房间内谁在线，以及谁正在输入。

功能点：

1. 展示在线成员列表。
2. 用户离开或断线后在线状态自动更新。
3. 用户输入时向同房间成员展示 typing 状态。
4. typing 状态不持久化。

验收标准：

1. 用户 A 和 B 同时进入房间时，双方都能看到对方在线。
2. 用户 B 退出后，用户 A 的在线列表在合理时间内更新。
3. 用户 A 输入但未发送时，用户 B 能看到 typing 提示。

### 6.5 股票 watchlist

每个房间有共享股票 watchlist，成员可查看关注股票。

功能点：

1. 展示房间 watchlist。
2. 支持添加股票代码。
3. 支持移除股票代码。
4. 支持按股票代码手动刷新。
5. 支持展示名称、最新价、涨跌额、涨跌幅、更新时间。

验收标准：

1. 成员添加 `AAPL.US` 后，同房间所有成员能看到 watchlist 更新。
2. 成员移除股票后，同房间 watchlist 同步更新。
3. 股票数据刷新失败时界面显示失败状态，不影响聊天。

### 6.6 股票信息展示

聊天界面内提供股票摘要和单只股票详情。

功能点：

1. 侧边栏展示 watchlist 报价摘要。
2. 支持命令 `/stock SYMBOL` 查看单只股票详情。
3. 支持消息中识别 `$SYMBOL`，并在本地提示可查看详情。
4. 股票信息包含价格、涨跌幅、更新时间和数据源。
5. 行情数据使用当前报价缓存，避免 terminal 客户端直接访问第三方 API。
6. 当前报价缓存只保存每个 symbol 的最新一条记录，不保存历史行情。
7. 缓存过期后由 Supabase Edge Function 触发刷新，刷新失败时可展示过期缓存和错误状态。

验收标准：

1. 输入 `/stock TSLA.US` 后展示 TSLA 当前缓存或触发刷新后的报价。
2. 股票 API 不可用时，返回明确错误，不泄露 API key。
3. watchlist 中多个股票刷新时不会阻塞聊天收发。
4. 同一个 symbol 多次刷新只覆盖当前缓存，不新增历史记录。

### 6.7 股票数据源和市场覆盖

第一版股票数据源选型为 Twelve Data。Twelve Data 覆盖美股、港股和 A 股，HTTP API 简单，适合放在 Supabase Edge Function 中作为缓存刷新服务。它的定位是展示型报价，不作为交易级实时行情源。

功能点：

1. 支持美股查询，例如 `AAPL.US`、`TSLA.US`。
2. 支持港股查询，例如 `0700.HK`、`9988.HK`。
3. 支持 A 股查询，例如 `600519.CN`、`000001.CN`。
4. 支持把用户输入的 symbol 标准化为内部 canonical symbol。
5. 支持在报价结果中展示 provider、市场、交易所、报价状态和更新时间。
6. 支持通过 adapter 替换 provider，不影响 terminal UI、数据库 watchlist 和聊天功能。
7. 支持配置报价缓存 TTL，MVP 默认 60 秒。

验收标准：

1. `/watch add AAPL.US` 可以添加美股。
2. `/watch add 0700.HK` 可以添加港股。
3. `/watch add 600519.CN` 可以添加 A 股。
4. Twelve Data API key 只存在于 Supabase Edge Function secret 中。
5. 将 provider 从 Twelve Data 切到另一个 adapter 时，terminal 命令和数据库 canonical symbol 不需要变化。

### 6.8 Terminal UI

应用第一屏就是可用的聊天界面，而不是介绍页。

功能点：

1. 主区域显示消息流。
2. 底部输入框发送消息或命令。
3. 右侧或顶部区域显示在线成员和股票 watchlist。
4. 支持键盘操作，不依赖鼠标。
5. 支持基础命令帮助。

MVP 命令：

```text
/start [nickname] [invite-code]
/logout
/rooms
/join <room>
/invite <nickname>
/invite-code
/members
/watch add <symbol>
/watch remove <symbol>
/stock <symbol>
/refresh [symbol]
/help
/quit
```

验收标准：

1. 用户可以只用键盘完成登录、进入房间、聊天、查看股票、退出。
2. terminal 窗口变窄时，消息内容不与输入框或状态栏重叠。
3. 输入未知命令时展示明确错误和 `/help` 提示。

## 7. 数据和隐私要求

1. 聊天消息只对房间成员可见。
2. 房间成员关系只对相关成员可见。
3. 股票 watchlist 只对房间成员可见。
4. 第三方股票 API key 不得出现在 terminal 客户端、本地配置文件或日志中。
5. Supabase service role 或 secret key 不得放入 terminal 客户端。
6. 所有暴露给 Supabase Data API 的表必须启用 RLS，并配置最小权限 grant。

## 8. 成功指标

1. 两个用户可以在 10 分钟内完成注册、进入同一房间并实时聊天。
2. 消息发送成功率在正常网络下达到 99% 以上。
3. 正常网络下实时消息延迟小于 2 秒。
4. 股票 watchlist 刷新失败不会影响聊天功能。
5. 手动验收覆盖登录、房间权限、聊天、在线状态、股票展示和退出登录。

## 9. MVP 里程碑

### M1：基础项目和登录

完成 Node.js TypeScript CLI、Supabase client、环境配置、登录和 session 持久化。

### M2：房间和聊天

完成 rooms、room_members、messages 表，完成 RLS，完成消息历史和实时收发。

### M3：在线状态

完成 Presence 在线状态和 typing 状态。

### M4：股票 watchlist

完成 room_watchlist、stock_quotes、Edge Function 股票代理和手动刷新。

### M5：定时刷新和 polish

完成可选的股票缓存预刷新、消息清理、错误状态、帮助命令和端到端验收。
