# @mentions

Date: 2026-06-25
Branch: `feature/mentions` (off `main`)

## Motivation

Let people @-mention a room member: pick the member when typing `@`, and make
both the mention text and "you were mentioned" visible.

## Decisions (from brainstorming)

- **Popup picker** (like the color/reorder panels): typing `@` in the composer
  opens a member picker above the input; `↑↓` select, `Enter` inserts `@<name> `,
  `Esc` cancels and inserts a literal `@`.
- **Trigger only at a word boundary** — `@` opens the picker only when it's at
  the start of the input or right after whitespace, so an in-word `@` (e.g. an
  email) just types a literal `@`. Also only when the room has members.
- **Plain-text storage, render-time highlight** — no DB / service / session
  change. The message body is plain text containing `@<name>`; highlighting is
  derived at render time from the current room members. Accepts that renames lose
  old highlights and same-named members both highlight (fine for a few friends).
- **Being mentioned: highlight only** (no bell/unread). The `@<name>` token is
  accent-colored for everyone; a message that mentions *me* also gets a leading
  accent gutter `▎` on my client.

## Architecture (entirely in `src/ui/`)

### Pure logic — `src/ui/mentions.ts` (Layer 1, strict TDD)

```ts
export type MentionSpan = { start: number; end: number; name: string };

// Find `@<member name>` runs in `text`. A match requires the `@` to sit at the
// start or right after whitespace, and the matched name to end at a non
// letter/digit boundary. Member names are matched longest-first so "Cool Cat"
// wins over "Cool" and "@alicea" does not match member "alice".
export function findMentionSpans(text: string, memberNames: string[]): MentionSpan[];

// Whether `text` @-mentions `name` (same matching). undefined/empty name -> false.
export function mentionsName(text: string, name: string | undefined): boolean;
```

### Composer — `src/ui/App.tsx`

- App-local state `mentionOpen: boolean`, `mentionIndex: number` (NOT session
  state — mentions are a pure composer/render concern).
- In `useInput`, before the normal composer editing path: if the typed character
  is `@`, the char before the cursor is start/whitespace, no other panel is open,
  and the room has members → open the mention picker instead of inserting.
- While `mentionOpen`: `↑↓` move `mentionIndex`; `Enter` inserts
  `@<selectedName> ` via `applyEditorAction(buffer, { type: 'insert', text })`
  and closes; `Esc` inserts a literal `@` and closes; `Ctrl+C` exits; other keys
  are swallowed (no type-to-filter — member counts are tiny). Reset
  `mentionIndex` to 0 when it opens.
- Render a `MentionPicker` panel above the input composer (reuse the color/reorder
  layout + height pattern; height = members + 4). Members come from the existing
  `selectMembers(state, roomId, …)`.

### Display — `src/ui/scroll.ts` + `MessageViewport`

- `buildRenderLines` gains an optional `mentionContext?: { memberNames: string[];
  selfName?: string }`. For each text message it computes `mentionsMe =
  mentionsName(body, selfName)`; for each wrapped row it computes
  `mentions = findMentionSpans(rowBody, memberNames)`. `RenderLine` gains
  `mentions?: MentionSpan[]` and `mentionsMe?: boolean`. Default (no context) =
  current behavior, so existing tests stay green.
- `MessageViewport` renders a row's body as segments: `@<name>` spans in an
  accent color (`cyanBright`), the rest plain. Rows of a `mentionsMe` message get
  a leading accent `▎` gutter so a mention of me stands out.
- `AppShell` passes `mentionContext` (member display names + my display name) to
  both the `MessageViewport` and the `buildRenderLines` height calc.

## Testing

- L1 strict TDD: `findMentionSpans` (plain, name-with-spaces, longest-first,
  word-boundary start, no match inside `email@host`, `@alicea` ≠ `alice`),
  `mentionsName`.
- L2 minimal: `MentionPicker` lists members; `MessageViewport`/buildRenderLines
  produces mention spans for a body containing `@name`.
- Smoke (`pnpm dev`): type `@` → picker → Enter inserts `@name ` → send → both
  clients show the `@name` highlighted; the mentioned person's client shows the
  `▎` gutter on that message; an in-word `@` (email) types a literal `@`.

## Out of scope

- Bell / unread / cross-room notification; `@everyone`; type-to-filter in the
  picker; structured mention metadata; any DB change.
